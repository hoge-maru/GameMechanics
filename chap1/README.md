## 1. ゲームメカニクスのデザイン

* ゲームはルールが重要
	* ルールに基づいて定量的な結果を得られる
* ゲームは予測不可能であるべき
	* 短いゲームなら偶然が支配する
	* 長いゲームなら意思決定やスキルも考慮する
	* ディプロマシー
		* プレイヤが他のプレイヤの選択を制御できない
		* ソーシャルスキルが重要
	* チェス
		* 各駒は単純だが組み合わさると複雑
		* 現在の状況を把握して戦略的な意思決定をすることが重要
	* 偶然、プレイヤの選択、複雑なルールの組み合わせ
	* 偶然の実装は簡単。複雑なルールはデザインが困難。
* ビデオゲームはゲームプレイを通してルールを伝えている
	* 説明書の内容がゲームルールで、プレイを通して伝えられることがゲームメカニクス
* モノポリーのメカニクス
	* 不動産の価格、チャンスカードと共同基金カードのテキストなど
* メカニクスは詳細化されていないとプログラマは実装できない
* コアメカニクス: 他のメカニクスと相互作用する大きな影響力のあるメカニクス
	* 例：重力のメカニクスは移動やジャンプに影響するのでコアメカクスである
	* 例：アイテムを取り出すメカニクスはコアメカニクスでない

* ゲームメカニクスは媒体に依存しない
	* ボードゲームでもビデオゲームでも同じメカニクスを実装できる
 	* フィジカルゲームとビデオゲームの複合のようなゲームも存在する
	* JohanSebastianJoust 	http://www.jsjoust.com/
	* ビデオゲームの実装は時間がかかるので、可能なのであれなボードゲーム等でのプロトタイピングがオススメ

* よく使用されるメカニクス
	* 物理、内部経済、進行型のメカニズム(キーアイテムをゲットしてエリアの開放など)
	* 戦術的な操作(ユニットの配置など)、社会的インタラクション(招待などのソーシャル要素)
* ゲームジャンル
	* RPGでは内部経済と進行が重要など、ゲームジャンル毎に重要なメカニクスが存在する
	* ゲームジャンル
		* アクション、ストラテジー、スポーツ、マネジメントシミュレーション、アドベンチャー、etc...

## 1.2 離散的メカニクスと継続的メカニクス

* 離散的メカニクス: 何かを獲得して結果を(整数で)得る
* 継続的メカニクス: 物理（小数単位で位置を調整できる）

* 継続的メカニクスはボードゲームで再現は困難。ビデオゲーム上では多くの演算を必要とする
* 過去のゲームは浮動小数点演算命令がなかったので物理は単純だった
* 離散的メカニクスは計画や戦略が立てられる
* AngryBirds
	* 正確な物理演算。鳥の種類や数の選択は離散的メカニクス。実行はタッチデバイスなので継続的メカニクス
* World of Goo
	* 知らないので省略。AngryBirdsより戦略的なゲーム。離散的メカニクスに依存
* 本書は離散的メカニクスに着目して解説している

# 1.4

* バーティカルスライス: ゲームの全要素を備えていて雰囲気が分かるプロトタイプ
* ホリゾンタルスライス: 一部の要素のみ。あるシーンの完全なUIのみなどのプロトタイプ。
* プロトタイプがあれば、それを捨てることになっても本製作で方向を見失わない
* プロトタイプではパラメータ等をデザイナがカスタマイズ可能にする
* spore 
	* www.spore.com/comm/prototypes
* ペーパープロトタイプ
	* ボードゲームで再現
	* トランプ、碁石(トークン)、ダイス(確率)、などの道具を準備
	* ソフトウェアで作るより素早く作れる。カスタマイズが容易。
* フィジカルプロトタイプ
	* サバゲーはFPSゲームの気持ちを体感できる
	* http://larp.meetup.com

* プロトタイプのフォーカス
	* 技術デモ
		* ゲーム技術で困難な部分に取り組んで作り上げられることを立証する
	* ゲーム内経済
		* 抜け道がないかなど。ペーパープロトタイプで素早く確認
	* インターフェース&操作系
	* チュートリアル

* 既存のゲームを参考にする
	* 開発規模が同じくらいのものを参考にする
