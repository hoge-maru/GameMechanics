GameMechanics
=============

GameMechanics


## 3.1-3.5

* 秩序と無秩序のバランスが重要
* テトリス
	* ランダムのピースと対処アクション
* 一貫性は重要
* 複雑度
	* 秩序： レベルデザイン
	* 周期系: 季節のサイクル、順番のターン
	* 創発: 日々の天候
	* 混沌: ダイスの目
* ネガティブフィードバック
	* 生態系のような捕食者と被捕食者のバランス
* ポジティブフィードバック
	* マイクのハウリングのように効果が増幅される
	* チェスは駒を取ると相手よりも駒が増えるので駒が取りやすくなる
* パックマン
	* 各モンスターの動作はシンプルだが、全体では追い込んでいるように動作する
* 鳥や魚の群れ
	* 固体間のフィードバックと固体と群れ全体の間のフィードバック
* 株価
	* 短期ポジティブフィードバックと長期ネガティブフィードバックの組み合わせ



## 4.1

* リソース
	* 数値で計測できるものすべて:お金、 時間、エネルギー、etc...
* リソースでない
	* 足場、壁、地形
* 有形リソース→無形リソースの変換
	* 救急キット→ライフ, 木→木材
* 抽象的リソース
	* "戦略的に優位である"という情報
* エンティティ
	* リソースを保存する場所、プログラマにとっては変数がエンティティ
* シンプルエンティティ
	* 1リソースだけのエンティティ
* 複合エンティティ
	* ヘルス、ダメージ能力、最大スピードなどを一まとめにしたエンティティ
* 経済メカニクス
	* ソース
		* リソースを生み出してエンティティに保存するメカニクス
	* ドレイン
		* エンティティからリソースを減らすメカニクス
	* コンバータ
		* あるリソースを別のリソースに変換するメカニクス
	* トレーダー
		* あるエンティティから別のエンティティにリソースを移動させるメカニクス

## 4.2-4.4

* 内部経済のグラフ
	* 成長していくグラフ
	* 増減を繰り返すグラフ
* チェスの内部経済
	* マテリアル：各駒にポイントを定義して、その総和
	* 戦略的に優位である、という抽象的リソース
	* マテリアルと戦略には関連がある
	* 序盤は定石に従う。中盤は戦略的優位性から仕掛ける。これらからおおまかなグラフの形が得られる
* ネガティブフィードバックでグラフを安定させる
* ポジティブフィードバックはインフレを作る
	* 軍拡競争
* デッドロック
	* 進行のための重要なリソースが敵に破壊されてゲーム進行ができなくなるなど
* ポジティブフィードバックを用いる場合は大きな差がついた時点でゲーム終了にする
* タンクラッシュ
	* 序盤で攻撃して敵を圧倒する戦略
* ネガティブフィードバックバスケット
	* 点差がつくごとに負けている側に選手追加
* ポジティブフィードバックバスケット
	* 点差がつくごとに買っている側に選手追加
	
* ラバーランド効果
	* レーシングゲームで遅れている場合にネガティブフィードバックが働く
* パンくず
	* FPSの弾薬、マリオのコインのような、プレイヤを誘導する収集可能なオブジェクト
* チェス、プリンスオブペルシャ
	* 駒/砂を管理してタイミングよく投資する
* RPGの内部経済
	* プレイヤのステータス上昇を選択できる場合、全部のステータスを最大値にしなければクリアできないようにするデザインはよくない
	* リプレイバリューを上げるために職業やクラスを設けるなど
	* レベルが上がると次のレベルまでに必要な経験値が増えるネガティブフィードバック
	* DeusEX
		* 戦闘、諜報、ハッキングなど、進行するための手段に選択肢がある
		* プレイヤのアバターがどういったステータスかわからないのでバランスをとるのが難しい
* 経済構築ゲーム
	* シムシティ
		* エリアに区画を割り当ててインフラを構築し、経済構造を構築する
	* 構成部品を一度にすべて導入しない(確率空間の制御)
	* メタ経済構造
		* 無限の組み合わせがあるようにみえて、特定の工業・商業・住宅区画の組み合わせが効率が良いなど
			* 長くその組み合わせを使うと公害が発生するなどして対処
		* 柔軟性のあるプレイヤーにリワードを与える
			* マップや災害の発生などで制御
			
## 4.5

* 演習1
	* Candy Crushのリソース
		* 各キャンディーの数、時間、移動回数、ライフ
	* Candy Crushの経済メカニクス
		* ソース：キャンディーを消すと別のキャンディーが生成される、
		* ドレイン:キャンディーを消す,移動して移動回数減、ゲームプレイでライフ減
		* コンバーター:課金でライフ回復
		* トレーダー：なし
* 演習2
	* ラバーランド効果
		* 湾岸ミッドナイトでCPU戦は一度遅れても離されないようになっている
	* 下方スパイラル
		* 塊魂
			* 塊を大きくしていくことが段々難しくなる
			* 塊を大きくするために必要なオブジェクトがフィールドから減っていく
		* CookieClicker
			* クッキー生成量を増やすことが難しくなる
			* クッキー工場などの生産オブジェクトを集めきった後に、さらに増やすための投資に必要なクッキーの生成量が多い
	* 短期投資、長期投資
		* ぷよぷよ
			* 短期で短い連鎖をしておじゃまぷよを送る
			* 長期で大きな連鎖をしておじゃまぷよを送る
			* 短い連鎖でリードをしても大連鎖に対抗できず窒息する
			* 長い連鎖を狙っても短い連鎖で発火点を潰されて窒息する
	* 周期的平衡のあるネガティブフィードバック
		* ドラクエでHPは宿屋に泊まると最大値まで回復し、敵と戦うと減る
	* プレイヤーの相対的スコアに基づいたフィードバック
		* 桃太郎電鉄
			* 1位のプレイヤーは皆から妨害される
				* 目的地までのマスの数
				* 収益・物件
				* 貧乏神
* 演習3
	* ロマンシングサガ３で王家の指輪やイルカの像を売ってもヤーマス武器屋で取り戻せる
